# 前端设计 Agent 输入文档 v2（全新 SaaS，中文界面）

## 1. 文档目的

本文件用于指导“前端设计 agent”直接产出教务管理 SaaS 的前端设计方案。  
重点输出：系统结构、页面功能、按钮与交互、中文文案规范、组件规范、交付格式。  

---

## 2. 产品定位

这是一个教务管理 SaaS，服务对象是培训机构（校长/教务/财务/老师）。  
系统目标：
1. 管理学员、老师、班级、课表、订单和财务；
2. 让日常操作高效、可追踪、可统计；
3. 提供清晰的中文界面与统一交互体验。

---

## 3. 全局要求（强约束）

1. **界面全中文**
2. **桌面端优先**，兼容中小屏。
3. **统一交互范式**：筛选区 + 表格区 + 分页区 + 详情侧栏/弹窗。
4. **统一按钮层级**：
   - 主按钮：`新增`、`保存`、`提交`、`确认`
   - 次按钮：`导出`、`导入`、`批量操作`
   - 弱按钮：`重置`、`取消`、`返回`
5. 所有列表页必须有：`加载态`、`空态`、`错误态`、`无权限态`。
6. 金额统一“元”显示，内部可按“分”处理；时间统一 `YYYY-MM-DD HH:mm`。

---

## 4. 用户角色与权限

至少支持 4 类角色：
1. `超级管理员`：全模块可见，可配置权限；
2. `教务`：学员/班级/课表/上课记录为主；
3. `财务`：订单/收支/课消记录为主；
4. `老师`：查看自己的课表、班级、上课记录。

权限要求：
- 页面级权限（能否进入页面）；
- 按钮级权限（能否点击新增/编辑/删除/导出）；
- 数据级权限（仅看本校区/本人相关）。

## 6. 页面功能与按钮清单（核心）

## 6.1 登录页
目标：账号安全登录，快速进入系统。

模块：
- 账号输入
- 密码输入
- 验证码（可选）
- 记住账号（可选）

按钮：
- `登录`
- `忘记密码`（链接）

交互：
- 登录成功跳转“工作台”
- 登录失败显示中文错误（例如：账号或密码错误）

---

## 6.2 工作台（Dashboard）
目标：一眼看到机构运营状态。

模块：
- KPI 卡片（学员总数、在读学员、本月订单、本月收入、本月消课）
- 今日待办（待确认订单、待点名课程）
- 快捷入口（新增学员、创建班级、创建课表、录入订单）

按钮：
- `刷新数据`
- `新增学员`
- `创建班级`
- `创建课表`
- `录入订单`

---

## 6.3 学员管理
目标：完成学员全生命周期管理。

列表字段建议：
- 学员姓名、手机号、性别、状态、所属顾问、最近上课时间、剩余课时

筛选项：
- 关键词（姓名/手机号）
- 学员状态（在读/停课/结课）
- 报名日期区间
- 所属班级

按钮：
- `新增学员`
- `导入学员`
- `导出学员`
- `批量分班`
- `重置筛选`
- 行内：`查看`、`编辑`、`报名/续费`、`停课`、`结课`

详情页/侧栏分区：
- 基础信息
- 报名订单
- 上课记录
- 课消记录
- 跟进备注

---

## 7. 通用组件规范（给设计与开发共用）

必须沉淀组件：
1. `页面头部`（标题 + 说明 + 右侧操作区）
2. `筛选栏`（输入框/下拉/日期 + 查询 + 重置）
3. `数据表格`（固定表头 + 横向滚动 + 空态）
4. `分页器`（上一页/下一页/跳页）
5. `详情侧栏`（标题 + 标签分区 + 底部操作按钮）
6. `状态标签`（成功/警告/错误/进行中）
7. `确认弹窗`（危险操作二次确认）
8. `消息提示`（成功/失败反馈）

危险操作必须二次确认：
- 删除
- 作废
- 退费
- 结班
- 停用

## 8. 中文文案规范（示例）

按钮文案：
- 新增、保存、取消、删除、编辑、查看、导出、导入、重置、查询、确认

状态文案：
- 加载中...
- 暂无数据
- 请求失败，请稍后重试
- 操作成功
- 操作失败，请检查后重试

字段文案统一：
- 学员姓名（不要混用“学生姓名”）
- 课消记录（不要混用“消课流水”）
- 收支明细（不要混用“流水总账”）
